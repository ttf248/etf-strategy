# 日线网格参数是怎么测试出来的

这份文档只解释日线主流程，也就是 README 里默认那套 `2024-01-01` 到 `2026-05-22` 的小米港股 `1810.HK` 回测。

## 先说结论

当前日线主流程的最优参数是：

- `grid_spacing = 7%`
- `grid_count = 5`
- `take_profit = 3%`

这不是“拍脑袋挑的”，而是程序把一组候选参数全部跑完后，用统一评分公式选出来的。

## 参数搜索到底怎么做

### 1. 先固定样本内和样本外

程序不会拿全量历史一起回测，而是先分两段：

- 样本内：专门用来找参数
- 样本外：专门用来验证参数是不是只对老数据有效

当前日线默认用 `2026-01-01` 作为样本外起点，所以：

- 样本外区间：`2026-01-01` 之后
- 样本内区间：`2026-01-01` 之前向前回看 `120` 天

这次日线主流程实际切出来的是：

- 样本内窗口：`2025-09-02` 至 `2025-12-31`
- 样本外窗口：`2026-01-01` 至 `2026-05-21`

### 2. 固定策略规则

每次测试参数时，策略骨架都不变，只换网格参数：

- 总资金：`200000`
- 样本开始时，用 `50%` 资金先建底仓
- 剩余 `50%` 资金专门留给网格
- 网格只卖出对应层，不会把底仓一起卖掉
- 所有下单都按“股数”执行，不按“金额”执行
- 港股会先查询最小交易单位，再按整手向下取整

### 3. 穷举所有候选参数组合

当前日线主流程会遍历这三组候选值：

- `grid_spacing`：`3% / 4% / 5% / 6% / 7%`
- `grid_count`：`4 / 5 / 6 / 7`
- `take_profit`：`3% / 5% / 7%`

总组合数是：

- `5 x 4 x 3 = 60` 组

也就是说，程序会把这 `60` 组全部跑一遍，不是随机抽样。

## 每组参数跑完后看什么

每组回测结束后，程序都会记录这些结果：

- `ReturnPct`
  - 整个账户最后赚了还是亏了多少。
- `MaxDrawdownPct`
  - 过程中最深回撤有多大。
- `EffectiveCost`
  - 把已经落袋的网格利润扣掉后，当前剩余持仓的等效成本。
- `CostReductionPct`
  - 相对样本起点底仓买入价，成本被摊低了多少。
- `RealizedGridProfit`
  - 已经完成“买入 -> 卖出”闭环、真正落袋的网格利润。
- `GridCyclesCompleted`
  - 完成了多少次网格闭环。
- `LotSize`
  - 这个标的最小交易单位是多少。
- `BaseUnits`
  - 样本起点底仓实际买了多少股。
- `GridUnitsPerLevel`
  - 每一层网格固定买多少股。

## 最优参数不是只看“谁收益最高”

当前项目的评分公式是：

```text
Score = ReturnPct - abs(MaxDrawdownPct) * 0.7 + CostReductionPct * 0.5
```

翻成人话就是：

- 收益越高越好
- 回撤越大越扣分
- 摊薄成本越明显越加分

这样做的原因很直接：

- 如果只看收益，容易选出“末尾反弹一点、但过程中回撤很深”的参数
- 如果只看成本摊薄，又可能选出“不断接飞刀、仓位越来越重”的参数

所以现在是把收益、回撤、成本下降三个目标放在一起评分。

## 为什么最后选到 `7% / 5 / 3%`

程序在 `outputs/optimize/in_sample_grid_search.csv` 里保存了 60 组结果，按评分从高到低排序。

最终最优参数是：

- 网格间距：`7%`
- 网格层数：`5`
- 止盈比例：`3%`
- 最小交易单位：`200` 股
- 底仓固定数量：`1600` 股
- 单层网格固定数量：`200` 股

它对应的样本内表现是：

- 样本内收益率：`-14.21%`
- 最大回撤：`19.51%`
- 期末有效持仓成本：`51.86`
- 成本下降：`7.15%`
- 网格已实现收益：`984.00`
- 网格循环次数：`3`

这里要特别注意：

- `grid_count = 5` 现在的意思不是“每层分 2 万预算”。
- 它的意思是：最多允许开启 `5` 层，每层都买固定 `200` 股。
- 之所以会得到 `200` 股，是因为程序先用样本起点价格估算“单层预算大约能买多少股”，再按港股 `200` 股一手向下取整。

要注意两件事：

1. 这个“最优”是相对当前候选参数集合和评分公式的最优，不代表放到别的年份也一定最优。
2. 这个“最优”也不代表绝对赚钱。它只是说明在这轮样本里，这组参数综合起来最不差。

## “用了网格，我到底赚了多少钱”怎么理解

最容易看错的是这两个数：

- `RealizedGridProfit`
- `ReturnPct`

区别是：

- `RealizedGridProfit` 只看已经完成闭环并落袋的网格利润
- `ReturnPct` 看的是整个账户最后总盈亏，包含底仓和仍在持有的网格浮盈浮亏

所以你会看到一种情况：

- 网格已实现收益是正的
- 但整个账户收益率仍然是负的

这不是矛盾，而是说明：

- 已经卖掉的网格单子确实赚了钱
- 但手里还没卖掉的仓位浮亏更大
