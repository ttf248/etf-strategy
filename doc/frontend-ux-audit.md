# 前端体验审查

本文记录面向“小白用户的回测平台”的前端页面审查结论。目标是把平台从运维控制台逐步调整为开箱即用的回测实验室。

## 总体判断

- 当前功能链路基本完整：数据准备、模板、提交回测、任务状态、报告列表和报告详情都有对应页面。
- 主要问题是信息架构偏运维：数据库、Worker、Scheduler、日志和模板管理过早暴露，用户还没完成第一次回测就需要理解平台内部概念。
- 第一轮调整已完成：导航改为“新手首页、创建回测、查看报告、数据准备、策略模板”，系统状态降级为维护入口；首页和回测页改为新手流程。
- 第二轮调整已完成：报告页增加解释型结论，数据准备页增加标的可回测检查，继续弱化内部表和进程概念。
- 第三轮调整已完成：策略模板页从参数后台改为模板库，高级编辑入口保留但不再作为新手主路径。
- 第四轮调整已完成：报告详情页继续翻译参数、交易和事件字段，不再把事件 JSON 作为默认阅读内容。
- 第五轮调整已完成：创建回测页增加策略选择卡片，用场景、风险和模板可用性替代单纯策略代码选择。
- 第六轮调整已完成：运行时逐页检查前端和关键 API，系统状态页在心跳表未初始化时给出 `init-db` 指引，后台进程不再持续刷 SQL 错误。
- 第七轮调整已完成：数据准备页的标的检查改为点击或回车触发，检查结果和覆盖表筛选同步更新。
- 第八轮调整已完成：截图检查发现移动端有 Ant Design Drawer 弃用告警导致开发浮层，已改用当前 API。
- 第九轮调整已完成：报告列表移动端改为卡片视图，保留结论、收益、回撤、交易次数和打开详情入口。
- 第十轮调整已完成：策略模板页移动端改为模板卡片，保留策略、周期、口径、状态、适用场景和操作；高级编辑 Drawer 同步改用当前 API，避免开发告警浮层。
- 第十一轮调整已完成：创建回测页移动端任务记录改为卡片，保留状态、进度、模板、错误说明和报告入口。
- 第十二轮调整已完成：数据准备页移动端覆盖记录改为卡片，保留周期、K 线数量、覆盖区间和最近更新。
- 第十三轮调整已完成：报告详情页移动端交易记录和事件流水改为卡片，常见报告字段转中文，减少横向表格和英文指标阅读成本。
- 第十四轮调整已完成：系统状态页改为维护检查页，默认显示健康摘要，进程、日志和调度折叠为高级维护信息。
- 第十五轮调整已完成：首页把同步和任务表折叠为高级数据明细，默认展示最近报告卡片和主路径入口。
- 第十六轮调整已完成：创建回测页从长表单改为三步向导，支持推荐模板选择和提交前确认。
- 第十七轮调整已完成：报告列表增加前端对比区，可选择多份报告对比收益、回撤和成交次数。
- 第十八轮调整已完成：数据准备页增加按当前标的一键同步入口，检查发现缺数据后可直接补数据。
- 第十九轮调整已完成：报告列表增加本地收藏、只看收藏筛选，并把收藏状态同步到移动端卡片、桌面对比区和表格操作列。
- 第二十轮调整已完成：数据准备页增加缺失周期推荐，直接告诉用户应补 1d、15m 还是 1m，并允许从推荐项一键同步。
- 第二十一轮调整已完成：首页增加“现成示例标的”，创建回测页支持从首页一键带入示例，并在进入下一步时自动套用推荐模板。
- 第二十二轮调整已完成：报告详情页增加“这几个数字怎么判断”解释区，把收益、回撤、买入持有对比、交易活跃度和下一步建议整理成阅读卡片。
- 第二十三轮调整已完成：首页增加“最近一次成功路径”，支持直接打开上次成功报告，或按相同标的、周期、策略和模板重回创建回测页。
- 第二十四轮调整已完成：模板页增加“推荐模板”区，并给推荐卡、移动端卡片和桌面表格都补上“去回测”入口，减少只会编辑不会使用的后台感。
- 第二十五轮调整已完成：报告详情页增加“回到报告列表”和“按当前配置重跑”入口，减少用户读完报告后还要手动回填配置。
- 第二十六轮调整已完成：系统状态页首屏进一步降级，增加“什么时候不用看这页”和按问题类型分流的维护路径卡，减少数据库、队列和进程信息的首屏干扰。
- 第二十七轮调整已完成：报告列表的对比区增加“对比后下一步”结论和直达动作，并给报告卡片、对比卡、桌面表格补上“按此配置重跑”入口。
- 第二十八轮调整已完成：补充 Playwright 主路径冒烟验证，默认复用本地 `3000` 前端或自动拉起测试实例，并用真实 API 验证首页到创建回测提交流程。
- 第二十九轮调整已完成：数据准备页增加新手示例标的推荐和当前库质量提示，把“先检查哪个标的、需不需要全量补数”直接讲清楚。
- 第三十轮调整已完成：报告详情页增加“去对比同标的报告”入口，报告列表支持通过 query 预带入当前报告和筛选条件，减少读完报告后再手工翻列表。
- 第三十一轮调整已完成：模板页增加“按目标找模板”和模板对比区，把“适合谁用、先试哪一个”直接说清楚。
- 第三十二轮调整已完成：首页增加“现在推荐你先做这一步”推荐卡，并给最近成功路径补上收益、回撤和成交情况摘要，减少用户回到首页后还要自己判断下一步。
- 第三十三轮调整已完成：报告详情页把“净值与回撤”区改成可读复盘区，新增曲线摘要、读图卡和图例说明，并把回撤线固定到 0% 下方。
- 第三十四轮调整已完成：全局壳层移除顶部数据库/时区/运行环境状态胶囊，改成基于当前页面的使用建议，并把侧栏分组改成“开始使用 / 排障时再看”。
- 第三十五轮调整已完成：报告列表桌面首屏从后台表格改成报告卡片库，对比空态也改成带动作的引导区，高级表格降级为折叠视图。
- 第三十六轮调整已完成：系统状态页首屏改成“要不要继续停留在这里”的判断页，并修正任务统计口径，把当前待处理任务和历史失败任务分开。
- 第三十七轮调整已完成：创建回测页把任务历史区改成“最近回测任务”摘要和最近任务卡片，完整历史与批量操作降级到高级折叠区。
- 第三十八轮调整已完成：数据准备页把完整覆盖表降级为高级明细折叠区，并把“当前数据准备提示”改成带动作的准备程度分层。
- 第三十九轮调整已完成：策略模板页把完整模板库和模板维护动作降级为高级管理折叠区，首屏优先服务“选模板去回测”。
- 第四十轮调整已完成：首页把推荐动作、示例标的和首页报告卡片的“为什么推荐”补成显式解释，不再只给动作入口。
- 第四十一轮调整已完成：报告列表把首屏排序依据显式化，默认按“收藏 -> 稳健正收益 -> 高波动正收益 -> 没成交 -> 反面对照”排序，并给每张卡片补上“为什么先看它”的解释。
- 第四十二轮调整已完成：报告详情页把参数字段墙改成“这套配置大概是什么意思”解释区，先翻译模板定位、交易节奏、仓位成本和重跑方向，再把完整参数降级到高级明细。

## 全局壳层审查

- 当前全局导航已从“控制台壳层”继续往“小白回测平台壳层”收敛：侧栏补了固定主路径提示卡，顶部不再展示数据库、时区和本地运行状态。
- 现在无论用户落在首页、报告页还是数据准备页，页头都会先告诉用户“这页最适合做什么”，而不是先暴露技术环境信息。
- 维护入口仍保留，但明确放到“排障时再看”分组，避免 `/platform` 和日常回测路径并列造成误导。

## 重构原则

- 首页只回答三个问题：能不能开始、怎么开始、结果在哪里看。
- 新手路径默认少填参数，复杂参数放到“高级”区域。
- 页面文案避免把数据库表、进程和内部模块作为主概念，除非用户进入系统状态页。
- 报告页优先解释结果含义，再展示底层指标。
- 数据页优先回答“这个标的能不能跑”，覆盖表只作为排查细节。
