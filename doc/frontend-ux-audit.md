# 前端体验审查

本文记录面向“小白用户的回测平台”的前端页面审查结论。目标是把平台从运维控制台逐步调整为开箱即用的回测实验室。

## 总体判断

- 当前功能链路基本完整：数据准备、模板、提交回测、任务状态、报告列表和报告详情都有对应页面。
- 主要问题是信息架构偏运维：数据库、Worker、Scheduler、日志和模板管理过早暴露，用户还没完成第一次回测就需要理解平台内部概念。
- 第一轮调整已完成：导航改为“新手首页、创建回测、查看报告、数据准备、策略模板”，系统状态降级为维护入口；首页和回测页改为新手流程。
- 第二轮调整已完成：报告页增加解释型结论，数据准备页增加标的可回测检查，继续弱化内部表和进程概念。
- 第三轮调整已完成：策略模板页从参数后台改为模板库，高级编辑入口保留但不再作为新手主路径。
- 第四轮调整已完成：报告详情页继续翻译参数、交易和事件字段，不再把事件 JSON 作为默认阅读内容。
- 第五轮调整已完成：创建回测页增加策略选择卡片，用场景、风险和模板可用性替代单纯策略代码选择。
- 第六轮调整已完成：运行时逐页检查前端和关键 API，系统状态页在心跳表未初始化时给出 `init-db` 指引，后台进程不再持续刷 SQL 错误。

## 页面审查

| 页面 | 当前可用性 | 主要问题 | 已处理 | 后续方向 |
| --- | --- | --- | --- | --- |
| `/` 新手首页 | 可读取统计、任务和报告 | 原先像数据库仪表盘 | 改成三步入口和准备情况 | 增加示例标的一键填充 |
| `/backtests` 创建回测 | 可提交任务、取消、重试 | 仍需要更强的一键推荐 | 拆成基础设置和高级参数；策略选择改为场景卡片，显示风险和当前周期模板可用性 | 做成真正分步向导，自动推荐模板 |
| `/reports` 查看报告 | 可筛选和打开报告 | 列表原先偏技术字段 | 增加简单结论、收益、回撤和解释文案 | 增加报告间对比和收藏 |
| `/reports/[id]` 报告详情 | 可看曲线、摘要、交易、事件 | 仍有少量指标需要解释 | 增加顶部结论卡片；翻译策略、参数、交易和事件；事件明细改为中文标签 | 增加指标解释和报告对比入口 |
| `/market-data` 数据准备 | 可查看覆盖和同步 | 原先像数据库覆盖表 | 增加标的可回测检查、周期卡片和补救同步入口 | 增加缺失周期的一键同步 |
| `/templates` 策略模板 | 可新建编辑模板 | 原先对新手过于专业 | 增加模板选择说明、策略适用场景和高级编辑提示 | 后续增加模板对比和推荐 |
| `/platform` 系统状态 | 可看服务、进程、日志 | 不应作为主入口；初始化问题需要可读提示 | 导航降级为维护入口；心跳为空时提示执行 `init-db` | 仅面向开发/运维说明用途 |

## 运行时检查记录

2026-05-27 本地检查：

- 前端路由 `/`、`/backtests`、`/reports`、`/market-data`、`/templates`、`/platform` 均返回 200。
- 后端接口 `/api/market-data/stats`、`/api/backtests?limit=5`、`/api/reports?limit=5`、`/api/templates?active_only=true`、`/api/platform/status` 均返回 200。
- 发现本地数据库缺少 `platform_heartbeats` 表时，Worker/Scheduler 会重复输出底层 SQL 错误；已改为一次性初始化提示。

## 重构原则

- 首页只回答三个问题：能不能开始、怎么开始、结果在哪里看。
- 新手路径默认少填参数，复杂参数放到“高级”区域。
- 页面文案避免把数据库表、进程和内部模块作为主概念，除非用户进入系统状态页。
- 报告页优先解释结果含义，再展示底层指标。
- 数据页优先回答“这个标的能不能跑”，覆盖表只作为排查细节。
