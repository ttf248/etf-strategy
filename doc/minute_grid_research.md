# Yahoo 分钟线支持情况与 15 分钟网格重算结果

这份文档回答三个问题：

1. Yahoo 到底支不支持分钟 K 线。
2. 当前项目怎么接入分钟线。
3. 这次用 `15m` 最近 `60d` 数据重算后，参数和回测结果是什么。

## 结论先说

- Yahoo 可以抓分钟 K 线。
- 当前项目已经支持通过配置和命令行切换分钟线。
- 这次默认采用：
  - 周期：`15m`
  - 数据范围：Yahoo 最近 `60d`
  - 样本切分：前 `75%` 做样本内，后 `25%` 做样本外

这次重算出来的分钟线最优参数是：

- `grid_spacing = 1.5%`
- `grid_count = 5`
- `take_profit = 1.0%`

## Yahoo 为什么能抓分钟 K 线

当前项目通过 `yfinance` 下载 Yahoo 数据。这个链路支持的分钟级周期包括：

- `1m`
- `2m`
- `5m`
- `15m`
- `30m`
- `60m`
- `90m`
- `1h`

但它有一个很重要的限制：

- 分钟级历史通常只能拿最近 `60` 天左右

这意味着：

- 日线主流程适合做更长区间的样本内 / 样本外研究
- 分钟线更适合做“最近一段市场微观波动”的短周期研究

## 项目里现在怎么切分钟线

默认配置在 `etf_strategy/config.py`：

- `DEFAULT_MINUTE_INTERVAL = "15m"`
- `DEFAULT_MINUTE_PERIOD = "60d"`
- `DEFAULT_MINUTE_DATA_PATH = data/processed/xiaomi_1810_hk_15m.csv`
- `DEFAULT_MINUTE_OUTPUT_DIR = outputs/minute`
- `DEFAULT_MINUTE_REPORT_DIR = reports/minute`

直接运行分钟线完整流程：

```powershell
py -3.13 main.py run --interval 15m --period 60d --proxy http://127.0.0.1:7897
```

如果只想基于已有分钟线 CSV 生成报告：

```powershell
py -3.13 main.py report --data data/processed/xiaomi_1810_hk_15m.csv --interval 15m
```

## 分钟线参数怎么测

分钟线这次没有沿用“`2026-01-01` 之前 / 之后”的切法，因为 Yahoo 免费分钟线没有那么长历史。

当前分钟线工作流改成：

1. 先下载最近 `60d` 的 `15m` 数据。
2. 按时间顺序切成：
   - 前 `75%`：样本内
   - 后 `25%`：样本外
3. 在样本内里找局部高点和 10% 回撤触发点。
4. 只在样本内穷举参数。
5. 把样本内最优参数直接拿到样本外验证。

当前分钟线候选参数范围是：

- `grid_spacing`：`1.0% / 1.5% / 2.0% / 3.0% / 4.0%`
- `grid_count`：`4 / 5 / 6 / 7`
- `take_profit`：`1.0% / 1.5% / 2.0% / 3.0%`

总共也是：

- `5 x 4 x 4 = 80` 组

评分逻辑和日线一致：

```text
Score = ReturnPct - abs(MaxDrawdownPct) * 0.7 + CostReductionPct * 0.5
```

## 这次 15 分钟线实际重算结果

### 样本切分

- 样本内窗口：`2026-03-19 02:30:00` 至 `2026-04-30 02:00:00`
- 样本外窗口：`2026-04-30 02:15:00` 至 `2026-05-21 08:00:00`

### 样本内最优参数

- 网格间距：`1.50%`
- 网格层数：`5`
- 止盈比例：`1.00%`

### 样本内结果

- 收益率：`-5.33%`
- 最大回撤：`8.44%`
- 期末有效持仓成本：`30.95`
- 成本下降：`6.78%`
- 网格已实现收益：`8599.82`
- 网格循环次数：`27`

人话解释：

- 这说明 15 分钟线下，策略成交更频繁，网格闭环次数明显多于日线。
- 但即便已经靠网格落袋了 `8599.82`，总账户还是没有回正，说明浮亏仍然存在。

### 样本外结果

- 收益率：`0.00%`
- 最大回撤：`0.00%`
- 触发建仓：`否`

人话解释：

- 不是分钟线报告算错了，而是这段样本外数据里根本没有出现“相对阶段高点回撤 10%”的进场条件。
- 所以策略没有出手，账户自然一直停在 `200000`，收益曲线也就保持 `0%`。

## 应该怎么理解这版分钟线结果

可以把它理解成：

- 日线主流程回答的是“这套网格在一轮中期下跌里大概什么样”
- 分钟线流程回答的是“最近 60 天的细颗粒波动里，这套网格会怎么成交”

它们不是互相替代关系，而是两条不同时间尺度的研究路径。

这次分钟线结果说明：

- 短周期下，网格成交会更密集
- 但样本外如果没有跌到建仓阈值，策略可以一直空仓
- 所以分钟线报告里出现 `0%` 收益并不奇怪
