# Yahoo 分钟线支持情况与 15 分钟网格重算结果

相关文档：

- [项目导航与阅读顺序](index.md)
- [术语表与口径说明](glossary.md)
- [回测报告阅读指南](report_reading_guide.md)

这份文档回答三个问题：

1. Yahoo 到底支不支持分钟 K 线。
2. 当前项目怎么接入分钟线。
3. 这次用 `15m` 最近 `60d` 数据重算后，参数和回测结果是什么。

## 结论先说

- Yahoo 可以抓分钟 K 线，但下载必须配置代理。
- 如果 Yahoo 下载失败，流程会直接停止并输出错误，不会切换到其他数据源。
- 当前项目已经把分钟线作为默认计算样本，也仍然支持通过 `--interval 1d` 显式切回日线。
- 这次默认采用：
  - 周期：`15m`
  - 增量下载窗口：Yahoo 最近 `60d`
  - 样本切分：前 `75%` 做样本内，后 `25%` 做样本外

这次重算出来的分钟线最优参数是：

- `grid_spacing = 4.0%`
- `grid_count = 4`
- `take_profit = 1.5%`

## Yahoo 为什么能抓分钟 K 线

当前项目优先通过 `yfinance` 下载 Yahoo 数据。这个链路支持的分钟级周期包括：

- `1m`
- `2m`
- `5m`
- `15m`
- `30m`
- `60m`
- `90m`
- `1h`

但它有两个很重要的限制：

- 分钟级历史通常只能拿最近 `60` 天左右
- 在批量下载或网络受限时，Yahoo 可能返回限流或 403

所以当前项目的处理方式不是“每次直接覆盖本地分钟线 CSV”，而是：

- 先下载这次能拿到的最近 `60d`
- 下载必须通过 `--proxy` 或 `ETF_STRATEGY_PROXY` 显式配置代理
- 如果 Yahoo 下载失败，直接停止流程并输出错误
- 再和本地已有分钟线样本按时间戳合并
- 重复时间戳以新数据覆盖旧数据

这意味着：

- 日线主流程适合做更长区间的样本内 / 样本外研究
- 分钟线更适合做“最近一段市场微观波动”的短周期研究

## 项目里现在怎么切分钟线

默认配置在 `etf_strategy/config.py`：

- `DEFAULT_MINUTE_INTERVAL = "15m"`
- `DEFAULT_MINUTE_PERIOD = "60d"`
- `DEFAULT_MINUTE_DATA_PATH = data/processed/1810_hk_15m.csv`
- `DEFAULT_MINUTE_OUTPUT_DIR = outputs/minute`
- `DEFAULT_MINUTE_REPORT_DIR = reports/minute`

直接运行分钟线完整流程：

```powershell
py -3.13 main.py run --period 60d --proxy http://127.0.0.1:7897
```

这个命令在进入回测前，会先把本次下载结果和本地 `data/processed/1810_hk_15m.csv` 合并。

如果只想基于已有分钟线 CSV 生成报告：

```powershell
py -3.13 main.py report --data data/processed/1810_hk_15m.csv --interval 15m
```

批量生成恒生科技成分股和 `513050.SS` 的分钟线报告：

```powershell
py -3.13 main.py batch --symbol-set hstech_plus_513050 --download --proxy http://127.0.0.1:7897 --jobs auto --cache-dir outputs/cache
```

## 分钟线参数怎么测

分钟线这次没有沿用“`2026-01-01` 之前 / 之后”的切法，因为 Yahoo 免费分钟线没有那么长历史。

当前分钟线工作流是：

1. 先通过代理下载 Yahoo 最近 `60d` 的 `15m` 数据。
2. 如果本地已经有历史分钟线 CSV，先按时间戳做增量合并。
3. 按时间顺序切成：
   - 前 `75%`：样本内
   - 后 `25%`：样本外
4. 再基于合并后的完整本地分钟线样本切分样本内和样本外。
5. 在样本内起点只记录网格锚定价，不直接买入。
6. 只在样本内穷举参数。
7. 把样本内最优参数直接拿到样本外验证。
8. 所有下单都按固定股数执行，并且先校验最小交易单位。
9. 默认同时计算 `hold` 和 `force_exit` 两套左侧处理口径。

当前分钟线候选参数范围是：

- `grid_spacing`：`1.0% / 1.5% / 2.0% / 3.0% / 4.0%`
- `grid_count`：`4 / 5 / 6 / 7`
- `take_profit`：`1.0% / 1.5% / 2.0% / 3.0%`

总共是：

- `5 x 4 x 4 = 80` 组

评分逻辑和日线一致：

```text
Score = ReturnPct - abs(MaxDrawdownPct) * 0.7 + ClosedGridReturnPct * 0.5
```

## 这次 15 分钟线实际重算结果

### 样本切分

- 样本内窗口：`2026-02-23 01:30:00` 至 `2026-04-30 02:00:00`
- 样本外窗口：`2026-04-30 02:15:00` 至 `2026-05-21 08:00:00`

### 样本内最优参数

- 网格间距：`4.00%`
- 网格层数：`4`
- 止盈比例：`1.50%`
- 最小交易单位：`200` 股
- 单层网格固定数量按纯现金网格预算和最小交易单位取整

### 样本内结果

- 报告会展示收益率、最大回撤、闭环网格净利润、未平网格浮动盈亏和强制退出是否触发。
- 如果左侧行情持续下跌，重点看 `hold` 和 `force_exit` 的差异，而不是只看闭环次数。

人话解释：

- 这说明 15 分钟线下，策略成交明显更频繁，网格闭环次数比日线多得多。
- 但即便已经靠网格落袋，总账户也可能没有回正，因为未平网格浮亏仍然存在。
- 这里的 `grid_count = 4` 意思是“最多开 4 层固定股数仓位”，不是“把预算拆成 4 份”。

### 样本外结果

- 收益率：`1.05%`
- 最大回撤：`5.21%`
- 样本外单层网格固定数量会按锚定价和最小交易单位重新取整
- 网格已实现收益可能为 `0.00`

人话解释：

- 如果样本外行情没有跌到新的网格买点，就不会产生网格成交。
- 纯现金网格没有初始持仓，因此“没有触发网格”时账户主要保持现金状态。

## 应该怎么理解这版分钟线结果

可以把它理解成：

- 日线主流程回答的是“这套网格在一轮中期窗口里大概什么样”
- 分钟线流程回答的是“最近 60 天的细颗粒波动里，这套网格会怎么成交”

它们不是互相替代关系，而是两条不同时间尺度的研究路径。

这次分钟线结果说明：

- 短周期下，网格成交会更密集
- 样本外如果没有新网格成交，策略本身不会因为初始持仓获得方向性收益
- 所以分钟线报告里如果出现“网格收益为 0”，需要先确认是否根本没有触发买入
