# 支持说明

## 先看哪里

提交问题前，建议先按下面顺序自查：

1. [README](README.md)：定位、安装、快速开始和常用命令。
2. [文档导航](doc/index.md)：架构、数据流、部署、运维和开发说明。
3. [贡献指南](CONTRIBUTING.md)：提交、测试和文档同步要求。
4. [开源准备度审计](doc/open-source-readiness.md)：了解当前仓库边界和已知缺口。

## 适合提 Issue 的情况

- 可复现的 Bug
- 文档与实际行为不一致
- 希望新增的功能或工程能力
- 与现有目录结构、配置组织或默认口径相关的改进建议

## 不适合提公开 Issue 的情况

- 安全漏洞：请按 [SECURITY.md](SECURITY.md) 走私下反馈。
- 仅与个人本地环境、私有数据或未提交脚本有关的问题，且仓库无法复现。

## 提问时请附带

- 运行命令
- 样例输入路径
- 关键日志或报错
- 操作系统、Python、Node.js 和 PostgreSQL 版本
