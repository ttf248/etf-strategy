# Codex 项目规则

## 语言与提交

- 文档、报告和面向用户的命令行输出默认使用中文。
- Git 提交日志必须使用中文，简洁描述本次变更目的。
- 每个独立任务完成后必须运行相关验证，并用中文提交。
- 如果检测到任务规模较大，必须先按可验证、可提交的粒度拆分子任务；每完成一个子任务就运行对应验证、提交 Git，然后继续下一个子任务。
- 拆分子任务时优先按功能边界划分，例如数据扫描、CSV 解析、公司行动抓取、前复权计算、MySQL 导入、绘图和文档测试分别处理。
- 每个子任务提交只包含该子任务相关文件，不混入本地输出目录、缓存文件或无关修改。

## 运行入口与依赖

- Python 程序以根目录 `main.py` 的 `main()` 函数作为启动入口。
- 不使用 `python -m 包名` 作为主要运行方式。
- 不在代码中硬编码 `.venv` 路径；VS Code Python 管理器负责选择解释器。
- 新增第三方库时必须同步维护 `requirements.txt`。

## 代码组织

- 新增命令或重要参数时必须同步更新 README 和测试。
- 代码注释默认使用中文，只解释关键业务规则、二进制格式、异常处理和不可从代码直接看出的设计取舍，不复述显而易见的代码流程。
- 新增或修改复杂模块时，应补充模块级说明、关键函数 docstring 或关键分支注释，确保后续维护者能理解数据口径、边界条件和失败处理策略。
