# 小米港股网格回测报告

## 摘要

- 标的：小米集团 `1810.HK`
- 数据周期：Yahoo Finance 最近 60 天 `15m`
- 样本内窗口：2026-03-19 02:30:00 至 2026-04-30 02:00:00
- 样本外窗口：2026-04-30 02:15:00 至 2026-05-21 08:00:00
- 切分方式：最近分钟线样本按 `75% / 25%` 拆分样本内与样本外
- 初始规则：从局部高点回撤 10% 后投入 50% 资金建底仓，剩余 50% 资金做网格买卖
- 最优参数：网格间距 1.50% / 网格层数 5 / 止盈比例 1.00%

在这轮样本里，网格交易能摊薄持仓成本，但还没有把总账户稳定拉回正收益。

## 样本内寻参结果

- 参考高点：2026-03-19 02:30:00，收盘价 37.14
- 初始建仓触发日：2026-03-20 08:00:00，收盘价 33.20
- 样本内收益率：-5.33%
  - 人话：整个账户从 `200000` 变成了 `189346.85`，合计亏损 `10653.15`。
- 样本内年化收益率：-38.90%
  - 人话：这是把这段样本期的结果折算成年化后的速度，用来和别的策略比较，不是说你真的持有满一年亏这么多。
- 样本内最大回撤：8.44%
  - 人话：样本里最差的时候，账户相对阶段最高点最多回撤了约 `8.44%`。
- 期末有效持仓成本：30.95
  - 人话：把已经通过网格落袋的利润扣掉后，你手里剩余持仓等效成本大约是 `30.95`。
- 相对初始建仓成本下降：6.78%
  - 人话：最开始底仓买在 `33.20`，现在等效成本被网格摊低了约 `6.78%`。
- 网格已实现收益：8599.82
  - 人话：单看“低买高卖已经完成并落袋”的网格交易，本轮一共赚了 `8599.82`。
- 完成网格循环次数：27
  - 人话：有 `27` 次网格仓位完成了“买入 -> 反弹 -> 卖出”的完整闭环。

### 样本内怎么看懂

- 如果你只按规则先买 `50%` 底仓，后面完全不做网格，到样本结束时账户大约是 `188072.48`。
- 当前这版网格策略的最终账户是 `189346.85`，收益率 `-5.33%`。
- 也就是说：网格本身虽然已经落袋赚了 `8599.82`，但额外接进来的下跌仓位浮盈浮亏也会影响总账户，所以整套策略相对“只拿底仓不做网格”多赚了 `1274.38`。
- 所以这里不能把“网格已实现收益”直接理解成“整个策略赚了这么多钱”；它只代表网格来回滚动已经兑现的那部分利润。

### 关于收益曲线为什么看起来像 0

- 图里的收益曲线不是全程为 0。
- 建仓前没有持仓，所以那一段收益率固定是 `0%`，这是正常的。
- 建仓后如果总账户没有回正，收益曲线就会从 `0%` 往下走；如果后来翻红，也会重新回到 `0%` 上方。
- 如果你肉眼看图时觉得它“都贴着 0”，主要是因为前半段建仓前确实是一条 0 线，而后半段始终在负区间，没有出现重新翻红。

## 结论

- 这套参数更适合“先急跌、后震荡修复”的行情，能够通过来回做网格降低持仓成本。
- 如果行情持续单边下跌，网格收益只能部分对冲亏损，不能替代趋势止损或更强的择时规则。
- 当前样本下，成本摊薄效果稳定存在：样本内下降 6.78%，样本外下降 0.00%。
- 这份报告只代表最近 60 天分钟级行情下的短周期表现，不等同于长期日线参数。
