## 按最小交易单位改为固定股数网格

### 状态

已完成，并已重算日线和分钟线报告。

### 修改方案

删除“每层网格按固定金额买入”的旧规则，改为：

- 先解析标的最小交易单位
- 底仓按整手向下取整
- 网格每层使用同样的固定股数

### 修改内容

- 最小交易单位规则：
  - 港股：抓取公开页面里的 `Lot Size`
  - 美股：默认 `1` 股
  - 其他市场：直接报错
- 命令和工作流：
  - `optimize` / `backtest` / `report` 支持显式传 `--symbol`
  - 不传时优先从 CSV 文件名推断
- 日线结果变化：
  - 最优参数从 `7% / 7 / 3%` 调整为 `7% / 5 / 3%`
  - 样本内底仓 `1600` 股，每层网格固定 `200` 股
  - 样本外底仓 `2400` 股，每层网格固定 `400` 股
- 分钟线结果变化：
  - 样本内底仓 `2600` 股，每层网格固定 `600` 股
  - 样本外底仓 `3400` 股，每层网格固定 `800` 股

### 设计规则、原因和收益

- 港股真实交易以整手为基本单位，继续按任意金额反推股数会让回测结果偏离实际可成交规则。
- 网格层数现在的含义更清晰：它表示“最多允许开几层固定股数仓位”，而不是“每层分多少钱”。
- 样本外固定股数可能和样本内不同，这是因为固定股数是按各自样本起点价格与整手规则重新计算出来的。

### 状态

进行中。当前已完成数据层与 CLI 下载口径改造，文档同步和默认日线样本刷新待后续子任务收尾。

### 修改方案

把当前“下载后直接覆盖”的数据链路改成：

- 分钟线优先和本地历史做增量合并
- 日线默认尽量抓 Yahoo 可提供的全历史

### 修改内容

- `etf_strategy/data/yahoo.py`：
  - 新增日线默认周期常量 `max`
  - 新增标准化行情合并逻辑，重复时间戳以新数据覆盖旧数据
  - `save_price_bars()` 支持按周期合并保存
  - 日线在未传 `start/end` 时自动走全历史口径
  - Yahoo Chart API 日线回退链路也支持无显式日期时抓全历史
- `etf_strategy/cli.py`：
  - `download` 日线命令改成“不传区间时默认全历史，传区间时必须同时传 `start/end`”
  - `run` 完整流程同步改成相同口径
  - 下载和完整流程在落盘 CSV 时默认先和本地已有样本做合并
  - 默认示例样本仍沿用仓库内置固定文件名，非默认标的或周期则按 `symbol + interval` 自动生成路径
- 新增测试：
  - `tests/test_yahoo_data.py`
  - 覆盖分钟线重叠时间合并、新数据覆盖旧数据、日线默认使用 `max`
  - `tests/test_grid_strategy.py` 增补 CLI 回归用例，覆盖“日线不传区间也能下载/运行”和“保存时默认增量合并”

### 设计规则、原因和收益

- 分钟线免费数据窗口只有最近 60 天，如果每次都覆盖，会把本地历史积累白白丢掉。
- 日线默认抓全历史后，后续新增其他策略时可以直接复用，不必再先补一轮基础数据下载策略。
- CLI 层只做“显式区间校验”，真正的“全历史口径”统一落在数据层，避免入口和底层逻辑再次分叉。
